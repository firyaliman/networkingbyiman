Project 1A & 1B · VLAN & Inter-VLAN Routing
Enterprise Office VLAN Segmentation
VLAN Trunking Inter-VLAN Cisco PT
⌄

Background

TechNova Sdn Bhd operates a single-site office with multiple departments sharing the same flat network, causing broadcast storms, security gaps, and poor traffic management.

Problem: All departments (HR, IT, Finance, Sales) were on the same subnet. Any broadcast hit every device. Sensitive finance data had no isolation.
Goal: Segment the network into department VLANs, enable inter-VLAN routing so they can still communicate where needed, and apply access controls.

Key Concepts Applied

  • 01 VLANs create logical network segments on the same physical switch — each department gets its own broadcast domain.
  • 02 Trunk links carry tagged frames for multiple VLANs between switches and routers using 802.1Q encapsulation.
  • 03 Router-on-a-Stick (ROAS) uses sub-interfaces on a single router port to route between VLANs.
  • 04 ACLs are applied to restrict which VLANs can reach sensitive subnets (e.g. Finance).

Project 2A · Routing & WAN
Multi-Site WAN Routing with OSPF & BGP
OSPF eBGP WAN ACL Cisco PT
⌄

Background

TechNova Sdn Bhd has expanded to three locations — a main HQ in Kuala Lumpur, Branch 1 in Penang, and Branch 2 in Johor. Each site has its own local network with department VLANs.

Problem: The three sites have no inter-site communication. Staff at branches cannot reach HQ resources or each other.
Solution: Deploy OSPF within each site, use eBGP between sites via ISP-simulated routers, and apply ACLs to restrict cross-site department access.

Key Concepts Applied

  • 01 OSPF (Area 0) — Link-state IGP used within each site to build a complete local routing table.
  • 02 eBGP — EGP used between sites, simulating how real ISPs connect autonomous systems.
  • 03 WAN Links — Serial point-to-point links between ISP routers simulate internet transit paths.
  • 04 Route Redistribution — OSPF routes redistributed into BGP so remote sites learn all local subnets.
  • 05 ACLs — Extended ACLs control which departments across sites can communicate.

HQ Router — OSPF + eBGP

router ospf 1 network 10.0.0.0 0.255.255.255 area 0 redistribute bgp 100 subnets ! router bgp 100 neighbor 203.0.113.2 remote-as 999 network 10.0.0.0 mask 255.0.0.0 redistribute ospf 1

Branch 1 (Penang) Router

router ospf 1 network 172.16.0.0 0.0.255.255 area 0 ! router bgp 200 neighbor 203.0.113.5 remote-as 999 network 172.16.0.0 mask 255.255.0.0 redistribute ospf 1

ACL — Cross-site Restriction
