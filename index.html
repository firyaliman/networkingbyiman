Project 1A · LAN Design & Switching
Hotel Campus Network — Dual-Core Redundancy & Segmentation
HSRP EtherChannel VLAN Segmentation Port Security OSPF (edge) Cisco PT

Background

TechNova Sdn Bhd operates a 3-story, high-occupancy hotel network supporting guest WiFi, guest room devices, hospitality IP phones, back-office administration, and point-of-sale payment terminals — all sharing the same physical infrastructure, with continuous uptime expected across every guest-facing and payment-processing system.

Problem: A single core switch meant one hardware failure could take down guest WiFi, phones, and payment processing at the same time. Guest devices, POS terminals, and staff systems had no traffic separation, so a compromised guest laptop could potentially reach payment infrastructure — a direct PCI-DSS compliance risk. On top of that, default switch settings left dynamic trunk negotiation (DTP) enabled and Native VLAN 1 active across every inter-switch link, leaving the backbone exposed to VLAN hopping.
Solution: Deploy two core multilayer switches in an HSRP-redundant pair, splitting the active gateway role per VLAN so both cores carry real traffic simultaneously instead of one sitting idle. Segment the network into purpose-built VLANs (Guest WiFi, Guest Rooms, POS, Voice, Admin, Management), bond access-switch uplinks with LACP EtherChannel, and run OSPF between the cores and a WAN edge router for uplink failover. Harden every trunk link by disabling DTP negotiation and moving native VLAN traffic onto an isolated, unrouted VLAN 999 dead-end network — closing off VLAN 1 as an attack surface. Layer ACLs to isolate guest traffic from POS/Admin, and port security to lock down user-facing ports.

Project 2A · Routing & WAN
Multi-Site WAN Routing with OSPF & BGP
OSPF eBGP Route Redistribution VLAN Hardening Cisco PT

Background

TechNova Sdn Bhd operates three sites — a Headquarters and two branch offices in Penang and Johor — each with its own local VLANs for users, servers, and management. The three sites needed to communicate reliably over WAN links simulating how real companies connect branch offices through an ISP.

Problem: Each site's local network was isolated with no path to reach the other two locations. Simply running one routing protocol everywhere wasn't realistic — real enterprises don't share a single IGP across sites they don't administer end-to-end, and don't want their entire internal topology exposed to an external network.
Solution: Run OSPF within each site to handle local inter-VLAN routing, and connect all three sites through a simulated ISP using eBGP — each site advertising its networks as its own Autonomous System. Routes are redistributed between OSPF and BGP at each site's edge router so local subnets become reachable across the WAN without merging the sites into one flat routing domain. Each site's switch also hardens its trunk links onto an isolated, unrouted VLAN 99 Black_Hole network to close off Native VLAN 1 as an attack surface.

What I Learned

Redistributing between OSPF and BGP at three separate edge routers made it clear why route redistribution needs to be handled carefully in a real network — each site is only supposed to know about the others' networks through BGP, not absorb their entire internal OSPF topology. Keeping OSPF and BGP as genuinely separate routing domains, only exchanging summarized reachability at the edge, is what actually makes a multi-site design scale without one site's internal changes rippling into another's routing table.
Project 3B · Security
Firewall Zoning & DMZ Architecture
ASA 5506-X DMZ Port Security DHCP Snooping DAI ACL Cisco PT

Background

TechNova Sdn Bhd needed to expose a small set of services — HTTP, email, and an external-facing FTP server — to outside users, while keeping internal resources like DHCP, internal FTP, NTP, and Syslog fully protected. Previously, no formal perimeter boundary existed between public-facing and internal resources, and internal switch ports had no protection against spoofing or rogue devices.

Problem: Public services and internal resources sat on the same trust level. If a public-facing server were compromised, an attacker would have a direct path into HR, IT, and management VLANs. On top of that, switch ports had no port security, and the network was open to Layer 2 attacks like DHCP spoofing and ARP poisoning.
Solution: Deploy a Cisco ASA 5506-X as a three-zone firewall — Inside (trusted LAN, including DHCP, Internal FTP, NTP, and Syslog services), DMZ (public-facing servers), and Outside (internet-facing). On the internal switches, layer port hardening, DHCP snooping, and Dynamic ARP Inspection (DAI) to stop Layer 2 attacks, and restrict SSH management access via ACL.
